# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

CineMatch is a collaborative movie watch party platform with real-time voting and ML-based recommendations. It consists of a Rust backend and a Next.js frontend, deployed as separate Docker containers communicating over HTTP/WebSocket.

## Development Commands

### Backend (Rust ‚Äî from `backend/`)

```sh
cargo build                                    # Build default member (cinematch-server)
cargo build --workspace                        # Build all crates
cargo run                                      # Run the API server
cargo run -p cinematch-importer                # Run the movie data importer CLI
cargo clippy --all-features -- -D warnings     # Lint (CI treats warnings as errors)
cargo fmt --all                                # Format (requires nightly toolchain)
cargo fmt --all -- --check                     # Check formatting without modifying
cargo test --workspace                         # Run all tests
diesel migration run                           # Run DB migrations (from cinematch-db/)
diesel migration revert                        # Revert last migration
```

Backend requires a `.env` file (copy from `.env.example`): `DATABASE_URL`, `REDIS_URL`, `QDRANT_URL`, `SECRET_TOKEN`, `RUST_LOG`.

### Frontend (Next.js ‚Äî from `frontend/`)

```sh
pnpm install       # Install dependencies
pnpm dev           # Start dev server (localhost:3000)
pnpm build         # Production build
pnpm lint          # ESLint
pnpm orval         # Regenerate API client from OpenAPI spec (requires backend running on :8085)
```

Frontend requires `.env.local` with `NEXT_PUBLIC_API_BASE=http://localhost:8085`.

### Infrastructure

```sh
# Start all backend services (PostgreSQL, Redis, Qdrant, Ollama, API server)
cd backend && docker-compose -f docker-compose.dev.yml up --build
```

## Architecture

### Backend Cargo Workspace (6 crates)

- **cinematch-server** (default member) ‚Äî Actix Web HTTP server, WebSocket handler, route definitions, auth (JWT + sessions). Organized by domain: `auth/`, `handlers/`, `movie/`, `party/`, `user/`, `recommendation/`.
- **cinematch-db** ‚Äî Diesel ORM models, schema, and migrations for PostgreSQL. Migrations live in `cinematch-db/migrations/`.
- **cinematch-common** ‚Äî Shared types and models used across crates.
- **cinematch-abi** ‚Äî ABI/interface definitions shared between crates.
- **cinematch-recommendation-engine** ‚Äî ML recommendation algorithms using linfa (collaborative filtering, content-based via Qdrant vectors, onboarding pool-based).
- **cinematch-importer** ‚Äî CLI tool (clap) for importing movie data, generating embeddings via Ollama, and populating Qdrant.

### Frontend (Next.js App Router)

- `src/app/` ‚Äî Pages: dashboard, create-party, party-room, preferences, API routes under `api/`.
- `src/server/` ‚Äî Server-side API client code, auto-generated by Orval from the backend's OpenAPI spec.
- `src/model/` ‚Äî TypeScript types auto-generated by Orval. **Do not edit `src/server/` or `src/model/` manually** ‚Äî regenerate with `pnpm orval`.
- `src/components/` ‚Äî React components (shadcn/ui, new-york style).
- `src/hooks/` ‚Äî Custom React hooks.
- `src/lib/` ‚Äî Utilities including the Orval custom fetch mutator (`orval-client.ts`).
- `src/actions/` ‚Äî Server actions.

### Data Flow

1. Frontend communicates with the backend REST API (`/api/*`) and WebSocket (`/ws/`).
2. Backend uses PostgreSQL (via Diesel) for persistent data, Redis for sessions (actix-session, 10-day TTL), and Qdrant for vector similarity search (1024-dim embeddings per movie).
3. The party system uses an Actix actor model for real-time WebSocket state management with phase transitions: Lobby ‚Üí Voting ‚Üí Watching ‚Üí Review (with configurable timeouts).
4. API documentation is auto-generated via utoipa and served at `/swagger-ui/`, `/redoc`, `/scalar/`, `/rapidoc/`.

## Code Style & Conventions

### Commits

Sentry-style conventional commits: `feat`, `fix`, `ref`, `perf`, `docs`, `test`, `build`, `ci`, `chore`, `style`. Subject line: imperative, capitalized, no period, max 70 chars. Branch naming: `<type>/<short-description>`.

### Rust

- Edition 2024, `rustfmt` style edition 2024 (nightly formatter).
- Clippy with all features, warnings as errors.

### Frontend

- TypeScript strict mode, path alias `@/*` ‚Üí `./src/*`.
- Prettier: single quotes, no semicolons, no trailing commas, 100-char width, LF endings.
- React Compiler enabled (babel-plugin-react-compiler).
- SWR for client-side data fetching, Zod for validation.
- Tailwind CSS 4 + shadcn/ui (new-york style, slate base color).
